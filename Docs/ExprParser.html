Description

  • exprD, exprF, exprI
  • ---------------------------------------
  • Family of Parsers for Integers, FLoats, and Doubles.
  • Method:
  • Used Parsec's Builtin Expr Helper Module To Construct Parser
  • Also, Ideas From Here Helped Construct This Parser.
  • Table is a List of Operators Along With Info on Their Associativities.
  • Factor is a Statement For Allowing The Parser to Try to Parse an Operator, Number or a Variable or Give an Error.
  • Format For Input:
  • Standard Ops are:
  • "(a+b)" => Add (a) (b)
  • "(a*b)" => Mult (a) (b)
  • "(a)^(b)" => Raise (a) (b)
  • "lna" => Natlog (a)
  • "(e**a)" => Exp (a)
  • "Cos a" => Cos (a)
  • "Sin a" => Sin (a)
  • "(x)" => Var (x)
  • "a" (a is a Number) => Const a
  • Brackets Are to Be used for Every Calculation With No Spaces Between The Operators. Also, Brackets Are to Be Put to Surround Whole Expressions That Are Being Multiplied, Added or Raised to other Expressions
  • ie. >>> parseExprI "(1+2)" (Add (Const 1) (Const 2))
  • >>> parseExprD "(((cos2.0)*(e**2.0))*(ln(x)))"
  • Mult (Mult (Cos (Const 2)) (Exp (Const 2))) (Natlog (Var "x"))
  • >>> parseExprF"(((1.0+2.0)^(x))*(2.0))"
  • Mult (Raise (Add (Const 1) (Const 2)) (Var "x")) (Const 2)
  • Seperate Parsing Functions For Double, Integer and Float Type Expressions:
  • parseExprI input => (Expr Integer)
  • parseExprD input => (Expr Double)
  • parseExprF input => (Expr Float)
  • input is a String in Format Specified Above
  • This Module Imports ExprType, ExprDiff and ExprPretty
